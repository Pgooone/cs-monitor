## 2026-04-15 - Task 5: 核心监控逻辑：波动分析与告警检测

### What was done:
- 创建 core/analyzer.py，实现 PriceAnalyzer 类
- 实现波动率计算：(当前价 - 基准价) / 基准价 × 100%
- 基准价优先使用 7 天均价（get_7day_average），缺失时回退到上一次采集价格
- 实现告警判断：涨幅 ≥ 阈值% 触发 price_surge，跌幅 ≤ -阈值% 触发 price_drop
- 实现告警冷却：同一饰品同一方向 4 小时内只告警 1 次
- 将触发的告警写入 alert_logs 表
- 集成 analyzer 到 monitor.py，每次采集后自动执行分析
- 为 database.py 添加 get_latest_price_any_platform() 支持无平台限制的基准价查询

## 2026-04-19 - Docker 部署实战与 WebSocket 问题修复

### Docker 部署过程记录

**Step 1: 首次构建失败 — Docker Hub 连接超时**
```
docker compose build
ERROR: failed to fetch anonymous token: dial tcp ...: connectex:
A connection attempt failed because the connected party did not
properly respond after a period of time
```
原因：中国大陆访问 Docker Hub 官方 registry 不稳定，无法拉取 `node:20-slim` 和 `python:3.12-slim` 基础镜像。

**Step 2: 配置国内 Docker 镜像源**
- 修改 `~/.docker/daemon.json`，添加 registry-mirrors：
  - `https://docker.m.daocloud.io`
  - `https://hub-mirror.c.163.com`
  - `https://mirror.ccs.tencentyun.com`
- 重启 Docker Desktop 使配置生效

**Step 3: 手动拉取基础镜像**
```bash
docker pull docker.m.daocloud.io/library/node:20-slim
docker tag docker.m.daocloud.io/library/node:20-slim node:20-slim
docker pull docker.m.daocloud.io/library/python:3.12-slim
docker tag docker.m.daocloud.io/library/python:3.12-slim python:3.12-slim
```
手动拉取并打 tag 后，docker compose build 可以复用本地已存在的镜像，跳过 registry 拉取。

**Step 4: 构建成功并启动**
```bash
docker compose build  # 成功，耗时约 3 分钟
docker compose up -d   # 容器启动，状态 healthy
```

**Step 5: 日志监控发现问题**
启动 `docker compose logs -f` 监控日志，发现以下警告频繁出现：
```
WARNING:  Unsupported upgrade request.
WARNING:  No supported WebSocket library detected.
Please use "pip install 'uvicorn[standard]'", or install
'websockets' or 'wsproto' manually.
```

**Step 6: 排查 WebSocket 问题**
- 问题根因：Task 22 生成的 Dockerfile 基于 `python:3.12-slim`，pip 安装的是基础版 uvicorn，不含 WebSocket 支持
- Task 19 实现 WebSocket 实时推送时，在本地 dev 环境测试通过（本地安装了 uvicorn[standard] 自动包含 websockets），但 requirements.txt 中遗漏了 `websockets` 依赖声明
- 导致 Docker 容器内 uvicorn 无法处理 WebSocket upgrade 请求

**Step 7: 修复并验证**
```
requirements.txt 添加：websockets>=12.0
docker compose down
docker compose build --no-cache
docker compose up -d
```
重建后重启容器，监控 2 分钟日志，WebSocket 警告彻底消失，`/ws/alerts` 连接正常建立。

### 部署验证结果
